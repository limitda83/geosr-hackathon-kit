# Before / After — 수작업 → 에이전트화

## 우리가 자동화한 반복 수작업
매일 고수온 뉴스 기사에서 지역명을 추출하고, 해당 지역 수온 데이터를 수집·저장한 뒤 분석 보고서와 경보를 제공하는 일련의 과정

## Before (기존 수작업 절차)
1. 뉴스 포털에서 "고수온" 관련 기사를 수동 검색·열람 (1~2시간)
2. 기사 본문을 직접 읽으며 지역명을 수기로 목록화 (30분)
3. 각 지역 위경도를 지도 앱·인터넷 검색으로 개별 확인 (30분)
4. 국가해양위성센터·KHOA 사이트에서 지역별 수온 파일을 개별 다운로드 (30~60분)
5. 엑셀/파이썬으로 데이터 정제·통합 (30분)
6. 시각화 차트·지도를 개별 제작 (30~60분)
7. 분석 결과를 Word/PPT로 수동 정리·경보 여부 판단 (1~2시간)
- **총 소요 시간: 1회 분석 시 약 4~6시간**

## After (에이전트화된 절차)
1. `main_agent.run_pipeline()` 한 번 실행 (또는 Streamlit 앱 접속)
2. news_agent → region-geocoder → ocean-data-collector → alert_agent → report_agent 순서로 자동 진행
3. PDF·Word 보고서 즉시 다운로드, 경보·주의보는 우상단 플로팅 토스트로 자동 알림
- **총 소요 시간: 3~5분 (자동화)**

## 효과 (자기 업무에 맞는 지표로 — 형식 자유)

| 지표(자기 업무에 맞게) | Before | After | 효과 |
|------|--------|-------|------|
| 1회 분석 소요 시간 | 약 4~6시간 (수작업) | 3~5분 (자동화) | 약 60~120배 단축 |
| 수온 데이터 수집 반복 횟수 | 6개 지역 수작업 개별 다운로드 | 자동 루프 수집 (92일 × 6개 지역 = 552건) | 반복 제거 |
| 경보 감지 누락 리스크 | 담당자가 수온 표 직접 확인 | 28°C 연속 조건 자동 판단 + 플로팅 알림 | 누락 제로화 |
| 보고서 일관성 | 작성자·날짜마다 포맷 다름 | 동일 템플릿으로 PDF+Word 자동 생성 | 형식 표준화 |
| 지역명 지오코딩 오류 | 수작업 검색 오타·좌표 오류 가능 | OSM Nominatim + 보정 로직 캐싱 | 오류율 감소 |

## Before (기존 초안의 인상)
- 기본 Streamlit 느낌이 강해서 페이지 간 톤이 조금 따로 놀았음
- 홈 화면이 정보는 있지만 첫 진입 인상이 다소 평평했음
- Metric, 카드, 버튼이 제각각이라 완성도가 덜 느껴졌음

## After (이번 디자인 개선)
- 전역 배경과 카드 스타일을 통일해서 전체가 하나의 서비스처럼 보이게 함
- 홈 화면에 그라데이션 히어로와 카드형 구조를 넣어 첫 인상을 강화함
- 공통 스타일을 모든 페이지에 적용해 발표용 대시보드 느낌을 살림

## 효과
| 지표 | Before | After | 효과 |
|------|--------|-------|------|
| 페이지 일관성 | 페이지별로 다름 | 전역 스타일 공유 | 통일감 상승 |
| 첫 화면 인상 | 평범한 대시보드 | 히어로 + 카드형 랜딩 | 발표 친화적 |
| 수정 난이도 | 각 페이지 따로 수정 | `utils/style.py` 중심으로 관리 | 유지보수 쉬움 |
