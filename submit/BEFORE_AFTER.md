# Before / After — 수작업 → 에이전트화

## 우리가 자동화한 반복 수작업
매일 고수온 뉴스 기사에서 지역명을 추출 → 해당 지역 수온 데이터 수집·저장 → 분석 보고서 작성까지 이어지는 일련의 과정

---

## Before (기존 수작업 절차)
1. 뉴스 포털에서 "고수온" 관련 기사를 수동 검색·열람 (1~2시간)
2. 기사 본문을 직접 읽으며 지역명을 수기로 목록화 (30분)
3. 각 지역 위경도를 지도 앱이나 인터넷 검색으로 개별 확인 (30분)
4. 국가해양위성센터·KHOA 사이트에서 지역별 수온 파일을 개별 다운로드 (30~60분)
5. 엑셀/파이썬으로 데이터 정제·통합 (30분)
6. 시각화 차트·지도를 개별 제작 (30~60분)
7. 분석 결과를 Word/PPT로 수동 정리 (1~2시간)
- **총 소요 시간: 1회 분석 시 약 4~6시간**

## After (에이전트화된 절차)
1. `main_agent.run_pipeline()` 한 번 실행 (또는 Streamlit 앱 접속)
2. news_agent → region-geocoder → ocean-data-collector → alert_agent → report_agent 순서로 자동 진행
3. PDF·Word 보고서가 생성되어 즉시 다운로드 가능, 경보·주의보는 플로팅 토스트로 자동 알림
- **총 소요 시간: 3~5분 (자동화)**

---

## 효과

| 지표 | Before | After | 효과 |
|------|--------|-------|------|
| 1회 분석 소요 시간 | 약 4~6시간 (수작업) | 3~5분 (자동화) | **약 60~120배 단축** |
| 지역 수온 데이터 수집 반복 횟수 | 6개 지역 × 수작업 개별 다운로드 | 자동 루프 수집 (92일 × 6개 지역 = 552건) | 반복 제거 |
| 경보 감지 누락 리스크 | 담당자가 직접 수온 표 확인 필요 | 28°C 연속 조건 자동 판단·플로팅 알림 | 누락 제로화 |
| 보고서 일관성 | 작성자·날짜마다 포맷 다름 | 동일 템플릿으로 PDF+Word 자동 생성 | 형식 표준화 |
| 지역명 지오코딩 오류 | 수작업 검색 오타·좌표 오류 가능 | OSM Nominatim + 보정 로직 캐싱 | 오류율 감소 |
| 시각화 제작 시간 | 매번 개별 제작 (30~60분) | Streamlit 앱 자동 렌더링 | 즉시 확인 |
